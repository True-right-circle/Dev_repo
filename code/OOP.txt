절차지향 vs 객체지향

절차지향(Procedure) -> 함수나 로직들을 순서대로 실행시키는 방법 
:함수의 호출 순서가 중요하다.
: 직관적인 장점이 있으나 프로그램의 규모가 커질수록 복잡하고 유지보수가 어렵다

객체지향(object oriented programming) -> 객체 지향 개발 방법
객체 : 속성(Data 해당 객체가 같은 값들 ), 기능(Method 해당 객체가 수행하는 기능들)

ex)
